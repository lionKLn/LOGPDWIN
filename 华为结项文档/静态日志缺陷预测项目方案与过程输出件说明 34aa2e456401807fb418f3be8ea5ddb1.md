# 静态日志缺陷预测项目方案与过程输出件说明

## 一、项目方案概述

本项目围绕“基于日志的缺陷预测与警报排序系统”展开，针对静态分析日志中误报率高、人工筛查成本大的问题，构建了一套融合代码结构、文本语义及结构化信息的缺陷预测模型，并在此基础上引入主动学习机制，实现模型的持续优化。

### 1.1 技术路线

系统整体流程如下：

```
原始日志数据（Excel）
        ↓
数据预处理与解析
        ↓
代码属性图构建（CPG）
        ↓
无监督图表示学习（GAE_GIN）
        ↓
多源特征融合（图+文本+结构化）
        ↓
缺陷预测模型（MLP）
        ↓
主动学习排序（新增）
        ↓
模型迭代优化
```

### 1.2 与中期方案的关系说明（重点）

### （1）中期方案（保持一致）

以下模块与中期方案保持一致：

### ① 代码属性图构建（CPG）

- 基于 AST / CFG / DDG 构建代码结构图
- 用于表达代码语义信息

### ② 无监督图表示学习（GAE_GIN）

- 对代码图进行表示学习
- 输出代码嵌入向量

### ③ 多源特征融合

- 代码嵌入（图）
- 文本嵌入（Desc / Func 等）
- One-hot 编码（结构化特征）

### ④ 缺陷分类模型（MLP）

- LogClassifier 二分类模型
- 输入 merged_features

### （2）本阶段新增与优化（重点）

### ① 主动学习排序机制（核心新增）

- 基于不确定性进行样本选择
- 多轮迭代训练

从：

```
一次性全量训练
```

变为：

```
初始训练 → 采样 → 增量训练 → 多轮优化
```

### ② 训练流程优化

- 支持 labeled / pool 划分
- 支持增量数据加入训练

### ③ 服务化接口设计

新增 FastAPI 服务：

- /encode
- /train
- /infer
- /sample

## 六、数据与模型关系说明

### 6.1 特征流程

```
Excel → 编码 → merged_features → pkl
```

### 6.2 模型依赖关系

```
模型 = 分类模型 + 编码器 + 图模型
```

需保证：

- 编码器一致
- 特征顺序一致
- 图模型一致

## 七、主动学习流程说明

### 7.1 流程

```
初始化 labeled / pool
        ↓
训练模型
        ↓
预测 pool
        ↓
不确定性采样
        ↓
加入 labeled
        ↓
重新训练
        ↓
循环
```

### 7.2 参数

```
rounds
query_size
sampling_strategy = uncertainty
```

### 7.3 优势

- 提高样本利用率
- 减少标注成本
- 提升模型性能

## 八、总结

本项目在中期提出的基于多源特征融合的缺陷预测模型基础上，保持代码图建模、无监督表示学习及分类模型等核心模块不变，引入主动学习排序机制，实现了模型训练方式由静态全量训练向动态迭代优化的转变。同时，通过服务化接口设计增强了系统的工程应用能力。

在过程输出方面，项目形成了结构清晰的代码工程体系，包括代码图构建模块、无监督训练模块、监督训练模块及新增的主动学习模块，并输出规范化的数据文件、模型文件及编码器文件，能够支持模型的训练、推理及持续优化，具备良好的工程复用价值。
