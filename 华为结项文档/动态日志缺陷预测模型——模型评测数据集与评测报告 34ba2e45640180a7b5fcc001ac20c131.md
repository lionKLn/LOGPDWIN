# 动态日志缺陷预测模型——模型评测数据集与评测报告

# 一、评测目标

本评测报告旨在验证动态日志缺陷预测模型在实际工程数据上的有效性，重点评估：

1. 模型对“误报 / 非误报”的分类能力
2. 特征融合方案的有效性（结构化 + 文本）
3. 模型在真实数据分布下的泛化能力

# 二、评测数据集

## 2.1 数据来源

数据来源于：

```
华为动态日志系统中提取的日志列表项数据
```

数据经过人工标注，形成监督学习数据集。

## 2.2 数据字段说明

本评测数据集包含如下字段：

| 字段名称 | 类型 | 说明 |
| --- | --- | --- |
| oracle_name | 类别 | 测试组件 |
| sut.component | 类别 | 系统组件 |
| sut.component_set | 类别 | 组件集合 |
| sut.module | 类别 | 模块 |
| api_ut | 文本 | API函数名 |
| tags | 文本 | 调用链 |
| false_positives | 标签 | 0/1 |

标签定义：

```
0：非误报（真实缺陷）
1：误报
```

## 2.3 数据预处理

数据在训练前进行了如下处理：

### （1）结构化特征处理

- 使用 OneHotEncoder 编码
- 未见类别自动忽略

### （2）文本特征处理

使用 CodeBERT：

```
api_ut → 768维向量
tags → 768维向量
```

### （3）特征融合

最终输入特征：

```
OneHot + CodeBERT(api_ut) + CodeBERT(tags)
```

## 2.4 数据划分

采用分层抽样：

```python
train : test = 8 : 2
stratify = y
```

确保：

- 类别分布一致
- 避免数据偏移

## 2.5 类别不平衡处理

采用：

```
RandomOverSampler
```

作用：

- 提升少数类（误报）比例
- 提高模型召回能力

# 三、模型配置

## 3.1 模型结构

分类器采用轻量级 MLP：

```python
Linear(input_dim → hidden_dim)
ReLU
Linear(hidden_dim → 2)
```

## 3.2 参数配置

| 参数 | 值 |
| --- | --- |
| hidden_dim | 128 |
| batch_size | 32 |
| epochs | 5 |
| optimizer | Adam |
| loss | CrossEntropyLoss |

## 3.3 训练策略

- 使用交叉熵损失函数
- 使用类别权重平衡样本
- 使用 DataLoader 批训练

# 四、评测指标

## 4.1 指标定义

本实验采用如下评测指标：

| 指标 | 说明 |
| --- | --- |
| Accuracy | 分类准确率 |
| Precision | 精确率 |
| Recall | 召回率 |
| F1 Score | 综合指标 |

## 4.2 关注重点

由于实际业务目标是：

减少误报，提高真实缺陷识别能力

因此重点关注：

```
Recall（召回率）
F1 Score（综合能力）
```

# 五、实验结果

## 5.1 整体性能

根据中期实验结果，模型在数据集上的表现如下：

### （1）训练集

| 指标 | 数值 |
| --- | --- |
| Accuracy | 82.24% |
| Precision | 0.84 |
| Recall | 0.80 |
| F1 | 0.82 |

### （2）测试集

| 指标 | 数值 |
| --- | --- |
| Accuracy | 81.3% |
| Precision | 0.99 |
| Recall | 0.81 |
| F1 | 0.89 |

## 5.2 结果分析

### （1）高精确率（Precision ≈ 0.99）

说明：

```
模型误报率极低
```

即：

被预测为“缺陷”的样本几乎都是真实缺陷

### （2）召回率表现良好（Recall ≈ 0.81）

说明：

```
模型能够覆盖大部分真实缺陷
```

### （3）F1 分数较高（≈0.89）

说明：

```
模型在精确率与召回率之间取得良好平衡
```

### （4）泛化能力稳定

训练集与测试集指标接近：

```
无明显过拟合
```

# 六、方法有效性分析

## 6.1 特征融合的作用

实验结果表明：

```
结构化特征 + 文本语义特征 → 显著提升模型能力
```

原因：

- OneHot 提供业务结构信息
- CodeBERT 提供语义信息

## 6.2 模型结构合理性

MLP 模型表现良好：

- 结构简单
- 收敛稳定
- 泛化能力强

说明：

当前特征表达已具备较强区分能力

## 6.3 类别不平衡处理效果

通过过采样：

```
提升了 Recall（缺陷检测能力）
```
