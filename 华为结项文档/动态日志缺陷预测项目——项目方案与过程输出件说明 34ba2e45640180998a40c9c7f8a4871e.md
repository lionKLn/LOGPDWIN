# 动态日志缺陷预测项目——项目方案与过程输出件说明

# 一、项目方案概述

本项目面向动态日志场景，设计并实现了一套基于文本语义与结构化特征融合的日志缺陷预测模型，用于自动识别日志中的误报与真实缺陷，从而提升缺陷检测效率并降低人工成本。

## 1.1 技术路线

系统整体流程如下：

```
动态日志数据（CSV）
        ↓
数据预处理
        ↓
特征编码（One-Hot + CodeBERT）
        ↓
特征融合
        ↓
分类模型（MLP）
        ↓
预测结果输出
```

# 四、核心模块说明

## 4.1 数据处理模块（preprocess）【中期一致】

- CSV读取
- 空值处理
- 字段清洗

## 4.2 特征编码模块【中期一致】

### （1）结构化特征编码

```
oracle_name
sut.component
sut.module
...
```

处理方式：

```

```

### （2）文本特征编码

字段：

```
api_ut
tags
```

处理方式：

```
CodeBERT → 768维向量
```

### （3）特征融合

```
OneHot + api_ut + tags
```

nn.Linear → ReLU → nn.Linear
