# 一、系统概述

本系统为动态日志缺陷预测模型系统，用于对华为动态日志中的列表项进行自动分类，识别日志是否为误报。

系统核心目标：

- 自动识别日志是否为误报（false_positive）
- 降低人工筛查成本
- 提高缺陷发现效率

## 1.1 模型特点

系统采用：

- 结构化特征 + 文本语义特征融合
- 基于 CodeBERT 的文本编码
- 轻量级 MLP 分类模型

模型输入：

```
One-Hot（结构化） + CodeBERT（文本） → 拼接特征 → 分类模型
```

## 1.2 系统整体流程

```
CSV日志数据
    ↓
数据预处理
    ↓
特征编码（One-Hot + CodeBERT）
    ↓
特征融合
    ↓
模型训练 / 推理
    ↓
输出预测结果
```

# 三、数据使用说明

## 3.1 数据格式

训练数据为 CSV 文件，必须包含以下字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| oracle_name | 类别 | 测试组件 |
| sut.component | 类别 | 系统组件 |
| sut.component_set | 类别 | 组件集合 |
| sut.module | 类别 | 模块 |
| api_ut | 文本 | 函数名 |
| tags | 文本 | 调用链 |
| false_positives | 标签 | 0/1 |

标签说明：

```
0 = 非误报（真实缺陷）
1 = 误报
```

（来自数据定义说明）

## 3.2 数据处理规则

- 文本字段不能为空
- tags使用空格分隔函数
- 类别字段自动One-Hot编码
- 文本通过CodeBERT编码

## 3.3 特征构成

最终输入特征：

```
One-Hot + CodeBERT(api_ut) + CodeBERT(tags)
```

# 四、模型训练

## 4.1 训练流程

```
CSV → 特征编码 → 数据划分 → 模型训练 → 模型保存
```

## 4.2 数据划分

```python
train:test = 8:2
stratify=y
```

保证类别分布一致

## 4.3 类别不平衡处理

使用：

```
RandomOverSampler
```

提升误报样本占比

## 4.4 模型结构

```python
nn.Sequential(
    nn.Linear(input_dim, hidden_dim),
    nn.ReLU(),
    nn.Linear(hidden_dim, 2)
)
```

特点：

- 轻量级
- 易部署
- 推理速度快

## 4.5 训练参数

| 参数 | 默认 |
| --- | --- |
| batch_size | 32 |
| epochs | 5 |
| hidden_dim | 128 |
| lr | 1e-4 |

## 4.6 启动训练

```bash
python train.py
```

## 4.7 输出文件

| 文件 | 说明 |
| --- | --- |
| log_classifier.pt | 模型 |
| encoder.pkl | OneHot编码器 |
| encoder_columns.npy | 特征顺序 |

# 八、使用注意事项

## 8.1 编码器一致性（非常重要）

必须保证：

```
训练模型 ↔ encoder.pkl ↔ encoder_columns.npy
```

否则：

- 特征维度错误
- 推理失败

## 8.2 环境一致性

必须保证：

```
训练环境 == 推理环境
```

否则：

- 模型无法加载
- 编码维度不一致

## 8.3 数据质量要求

- 无空值
- 字段完整
- 格式统一
