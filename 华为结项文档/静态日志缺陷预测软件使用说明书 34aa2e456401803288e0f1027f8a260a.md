# 一、系统概述

本系统为一个基于日志的缺陷预测与主动学习排序服务系统，用于解决大规模静态分析日志中误报多、人工筛查成本高的问题。

系统提供从数据处理到模型优化的完整闭环能力：

```
日志数据 → 特征编码 → 模型训练 → 模型推理 → 主动学习采样
```

系统基于 FastAPI 构建服务接口，通过 HTTP 调用完成核心功能。

## 1.1 系统核心能力

- 多源特征融合（代码 + 文本 + 结构化信息）
- 缺陷预测模型训练与推理
- 主动学习驱动的样本筛选
- 编码器与模型绑定机制（关键设计）

# 六、接口说明

## 6.1 编码接口 /encode

### 功能说明

将 Excel 数据转换为模型输入特征（pkl），并管理 One-Hot 编码器。

### 请求方式

```
POST /encode
```

### 参数

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| file | 是 | Excel 文件 |
| mode | 是 | train / infer |
| encoder_tag | train可选 / infer必填 | 编码器标识 |

### 模式说明

### 🔹 train 模式

- 生成训练数据 pkl
- 自动生成编码器

生成文件：

```
{encoder_tag}_onehot_encoder.pkl
{encoder_tag}_onehot_feature_names.npy
```

### 🔹 infer 模式

- 必须提供 encoder_tag
- 使用已有编码器

### 核心机制（重点）

```
encoder_tag 是编码器与模型绑定的唯一标识
```

### 返回示例

```json
{
  "output_pkl_filename": "...",
  "encoder_tag": "...",
  "note": "模型必须绑定该 encoder_tag"
}
```

## 6.2 训练接口 /train

### 功能说明

该接口用于基于训练数据执行主动学习训练流程，完成模型训练并生成最终模型文件。

训练过程包括：

```
1. 数据加载与划分（labeled / pool / test）
2. 主动学习循环训练
3. 模型保存
```

系统通过主动学习策略，从未标注数据池中逐步选择高价值样本进行迭代训练，从而提升模型性能并降低标注成本。

### 请求方式

```
POST /train
```

### 请求参数（JSON）

```json
{
  "pkl_filename": "xxx_train_xxx.pkl",
  "rounds": 3,
  "query_size": 100,
  "sampling_strategy": "uncertainty",
  "test_size": 0.2,
  "init_ratio": 0.1,
  "seed": 42,
  "dedup_by_id": true
}
```

### 参数说明（重点）

### 1. pkl_filename

```
训练数据文件名（必须）
```

说明：

- 必须为 /encode 接口在 train 模式下生成的 pkl 文件
- 文件中必须包含：

```
merged_features
false_positive（标签）
```

作用：

- 作为模型训练的输入数据源

### 2. test_size

```
测试集比例（默认 0.2）
```

作用：

- 从数据集中划分一部分作为测试集
- 用于评估模型性能（如 acc / f1）

### 3. init_ratio

```
初始 labeled 数据比例（默认 0.1）
```

作用：

- 初始训练时使用的数据比例
- 剩余数据作为未标注池（pool）

示例：

```
10000条数据 → 1000 labeled + 9000 pool
```

### 4. seed

```
随机种子（默认 42）
```

作用：

- 保证数据划分可复现
- 保证实验稳定性

### 5. dedup_by_id

```
是否去重（默认 true）
```

作用：

- 按样本 ID 去除重复数据
- 避免训练数据冗余影响模型

### 6. rounds

```
主动学习迭代轮数
```

作用：

- 控制主动学习循环次数
- 每一轮都会：

```
训练 → 采样 → 扩充 labeled → 再训练
```

### 7. query_size

```
每轮采样样本数
```

作用：

- 每一轮从 pool 中选取多少样本加入 labeled 集

示例：

```
round=3, query_size=100 → 共新增300条标注数据
```

### 8. sampling_strategy

```
采样策略（当前支持：uncertainty）
```

当前实现：

```
uncertainty（不确定性采样）
```

原理：

```
选择预测概率最接近 0.5 的样本
```

即模型最“拿不准”的样本

作用：

- 优先标注最有价值的数据
- 提升训练效率

### 内部执行流程（结合代码说明）

### Step 1：数据加载与划分

```python
X_labeled, X_pool, y_labeled, y_pool, X_test, y_test = load_and_split_active_learning(
    pkl_path=str(pkl_path),
    test_size=req.test_size,
    init_ratio=req.init_ratio,
    seed=req.seed,
    dedup_by_id=req.dedup_by_id
)
```

功能：

- 加载 pkl 数据
- 去重（可选）
- 划分：

```
labeled / pool / test
```

### Step 2：主动学习训练

```python
model, history = active_learning_loop(
    X_labeled,
    X_pool,
    y_labeled,
    y_pool,
    rounds=req.rounds,
    query_size=req.query_size,
    sampling_strategy=req.sampling_strategy,
    mode_save_path=str(output_model_path)
)
```

每一轮执行：

```
1. 使用 labeled 数据训练模型
2. 在 pool 中进行预测
3. 根据策略选取样本（uncertainty）
4. 将样本加入 labeled
5. 更新模型
```

### Step 3：模型保存

```
保存路径：
server/data/models/
```

### 返回结果

```json
{
  "output_model_filename": "xxx_trained_model.pt"
}
```

### 关键约束（非常重要）

### 1. 数据来源约束

```
pkl_filename 必须来自 /encode(train)
```

错误：

```
使用 infer pkl 训练
```

### 2. 编码器绑定关系

```
pkl_train → encoder_tag → model
```

推理时必须使用相同 encoder_tag

### 3. 采样策略限制

当前支持：

```
uncertainty
```

### 常见错误

### 数据无标签

```
原因：使用 infer 数据训练
```

### 维度不一致

```
原因：编码器不匹配
```

### 训练效果异常

```
原因：
- 数据重复
- init_ratio过小
- query_size过大
```

### 小结

训练接口通过主动学习机制，实现了从初始少量标注数据出发，逐步扩展训练集并优化模型性能的过程。相比传统一次性训练方式，该方法能够有效降低标注成本并提升模型效果，适用于大规模日志分析场景。

# 八、核心设计说明（重点）

## 模型与编码器绑定机制

### 设计目的

解决：

```
训练与推理特征不一致问题
```

### 绑定关系

```
encoder_tag = 唯一标识

train:
    Excel → pkl → encoder_tag → model

infer:
    Excel → (同 encoder_tag) → pkl → model
```

### 关系图

```
encoder_tag_A
    ├── onehot_encoder.pkl
    ├── feature_names.npy
    └── model.pt
```

### 核心结论

```
模型必须使用对应 encoder_tag
```
