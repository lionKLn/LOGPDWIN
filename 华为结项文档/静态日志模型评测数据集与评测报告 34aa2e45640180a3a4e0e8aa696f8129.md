# 静态日志模型评测数据集与评测报告

# 一、评测目标

本报告旨在对所设计的基于日志的缺陷预测模型及主动学习排序策略进行系统评估，验证以下两点：

1. 基于多源特征融合的 MLP 分类模型具备有效的缺陷识别能力；
2. 基于不确定性采样的主动学习策略，相较于随机采样，能够更高效地提升模型性能。

# 二、评测数据集说明

## 2.1 数据来源

本实验数据来源于实际工程中的静态分析日志数据，包括：

- 代码片段（code_str）
- 日志描述（Desc）
- 结构化字段（component、rule 等）

## 2.2 数据规模

```
总样本规模：约 1w 条
初始标注集（Labeled）：约 906 条
未标注数据池（Pool）：约 8161 条
独立测试集（Test）：约 2267 条
```

## 2.3 标签定义

```
false_positive = 1 → 误报
false_positive = 0 → 真实缺陷
```

## 2.4 数据划分策略

- 使用 `train_test_split` 划分训练集与测试集
- 在训练集中进一步划分：
    - 初始 labeled 集
    - 未标注 pool 集

# 三、模型与方法说明

## 3.1 特征表示方法

本系统采用多源特征融合，包括：

- 代码结构特征（CPG 图嵌入）
- 文本语义特征（Sentence-BERT）
- 结构化特征（One-hot 编码）

最终融合为统一特征向量 `merged_features`

## 3.2 分类模型（MLP）

采用单隐藏层多层感知机（MLP）作为分类器：

```
输入：merged_features
结构：Linear → ReLU → Dropout → Linear → Softmax
输出：二分类概率
```

## 3.3 主动学习策略

本系统采用：

### 不确定性采样（Uncertainty Sampling）

策略定义：

```
选择预测概率最接近 0.5 的样本
```

即模型最不确定的样本。

## 3.4 对比方法（Baseline）

采用：

### 随机采样（Random Sampling）

从未标注数据池中随机选取样本进行训练。

# 四、实验设置

## 4.1 主动学习流程

```
1. 使用初始 labeled 数据训练模型
2. 在 pool 上进行预测
3. 根据采样策略选择样本
4. 加入 labeled 集
5. 重新训练模型
6. 重复迭代
```

## 4.2 实验参数

```
主动学习轮数：10
每轮采样数量：200
初始 labeled 比例：10%
```

## 4.3 评价指标

采用以下指标：

- Accuracy（准确率）
- Recall（召回率）
- F1-score（综合指标）
- Precision（精确度）

# 五、实验结果与分析

## 5.1 不确定性采样结果

随着主动学习轮数的增加，模型性能逐步提升，F1-score 持续增长，说明模型能够通过引入高价值样本不断优化分类能力。

## 5.2 随机采样结果

随机采样在初期能够提供一定性能提升，但随着迭代进行，模型性能波动较大，整体提升幅度有限。

## 5.3 方法对比

与随机采样相比，不确定性采样在相同标注成本下，能够取得更高的 F1-score 和更稳定的性能表现。

# 六、关键结论

## 6.1 分类模型有效性

实验结果表明，基于多源特征融合的 MLP 模型能够有效识别缺陷日志，在测试集上取得较高的召回率与准确率，说明模型具备实际应用价值。

## 6.2 主动学习策略有效性

不确定性采样策略显著优于随机采样，能够优先选择对模型最有价值的样本进行标注，从而在有限标注成本下提升模型性能。

## 6.3 工程意义

该方法能够减少人工标注成本，并提升缺陷识别效率，适用于大规模日志分析场景。
